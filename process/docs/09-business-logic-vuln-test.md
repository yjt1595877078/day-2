# 步骤 9 — 业务逻辑及越权漏洞测试

> **测试日期:** 2026-07-09  
> **测试功能:** 个人中心 + 充值  
> **漏洞类型:** 越权访问 / 负金额充值 / ID枚举

---

## 一、测试范围

新增的个人中心 `/profile` 和充值 `/recharge` 功能未做任何权限校验和参数验证。

---

## 二、漏洞详情

### 🔴 漏洞 1：无需登录访问个人中心

**测试方法：** 不带任何 cookie 直接访问

```bash
curl "http://.../profile?user_id=admin"
```

**结果：** 无需登录即可查看任意用户的资料（邮箱、手机、余额）。

**风险：** 未登录用户可批量抓取所有用户资料。

---

### 🔴 漏洞 2：水平越权（查看他人资料）

**测试方法：** admin 登录后修改 `user_id` 参数

```bash
curl -b cookies.txt "http://.../profile?user_id=alice"
```

**结果：** admin 可以查看 alice 的完整资料。

**风险：** 任意用户可查看其他用户的个人信息。

---

### 🔴 漏洞 3：水平越权（给他人充值）

**测试方法：** admin 登录后修改 `user_id` 参数充值

```bash
curl -X POST -b cookies.txt -d "user_id=alice&amount=9999" "http://.../recharge"
```

**结果：** admin 可以给 alice 的账户充值。

**风险：** 任意用户可操作其他用户的余额。

---

### 🔴 漏洞 4：垂直越权（普通用户查看管理员）

**测试方法：** 注册普通用户 `newuser1`，登录后查看 admin

```
普通用户 newuser1 → /profile?user_id=admin → 成功查看
```

**结果：** 普通用户可查看管理员资料。

**风险：** 权限分级完全失效。

---

### 🔴 漏洞 5：负金额充值（套现）

**测试方法：**

```bash
curl -X POST -d "user_id=admin&amount=-99999" "http://.../recharge"
```

**结果：** 余额被扣减，可无限透支。

**风险：** 攻击者可恶意扣减任意用户余额。

---

### 🟡 漏洞 6：余额无下限

充值负金额后余额变为负数，无最低余额限制。

---

### 🟡 漏洞 7：充值无上限

```bash
amount=999999999999999
```

充值金额无上限，可导致金额溢出。

---

### 🟡 漏洞 8：用户 ID 可枚举

```bash
curl "http://.../profile?user_id=1"  # 用户存在
curl "http://.../profile?user_id=99" # 不存在
```

遍历 `user_id` 可探测系统存在的用户。
