## 二、漏洞测试结果

### 🔴 漏洞 1：路径穿越（目录遍历）

**测试方法：** 上传文件时使用包含 `../` 的文件名

```bash
curl -F "file=@test.txt;filename=../../../evil.txt" http://.../upload
```

**结果：** 文件成功逃逸出 `uploads/` 目录

```
原目标: process/static/uploads/evil.txt
实际保存: process/evil.txt          ← 路径穿越成功！
```

**风险：** 攻击者可覆盖任意系统文件（如 `/etc/crontab`）。

---

### 🔴 漏洞 2：任意文件上传（无类型检查）

**测试 payload：**

| 文件类型 | 结果 |
|----------|:----:|
| `xss.html`（含 `<script>`） | ✅ 上传成功 |
| `shell.php` | ✅ 上传成功 |
| `test.py` | ✅ 上传成功 |
| `evil.js` | ✅ 上传成功 |
| 无扩展名文件 | ✅ 上传成功 |

**风险：** 任何文件类型均可上传，无法阻止恶意文件。

---

### 🔴 漏洞 3：持久化 XSS

**测试方法：** 上传含 JavaScript 的 HTML 文件

```bash
curl -F "file=@xss.html" http://.../upload
访问 http://.../static/uploads/xss.html
```

**结果：** HTML 中的 `<script>` 在浏览器中直接执行。

**风险：** 攻击者可上传钓鱼页面，诱导其他用户访问。

---

### 🟡 漏洞 4：文件覆盖

**测试方法：** 两次上传同名文件

**结果：** 后上传的文件直接覆盖前者，无任何提示。

**风险：** 可覆盖他人的合法上传文件。

---

### ✅ 已有防护

| 防护项 | 说明 |
|--------|------|
| `MAX_CONTENT_LENGTH = 16MB` | 超过限制的文件被自动拒绝 |
| Flask 不支持 PHP 解析 | PHP 木马上传后无法执行代码 |

---

## 三、根本原因

所有漏洞源于同一个问题：**对用户上传的文件未做任何校验**。

```python
# 当前代码（无任何检查）
file.save(os.path.join(upload_dir, filename))

# 需要修复的方向：
# 1. 过滤文件名中的 ../ 路径穿越
# 2. 限制允许的文件类型（如仅图片）
# 3. 使用 UUID 重命名文件
```
